# 回合制格斗游戏 — 设计总文档

## 1. 游戏概述

- **类型**：2D 像素横版、回合制、放置型策略格斗
- **平台**：网页游戏（浏览器运行），**适配手机和电脑**
- **角色**：仅一个角色——**骑士**（持剑和盾），双方玩家使用相同角色
- **核心玩法**：选技能 → 回合制移动+攻击 → 耗尽对方血量获胜
- **总技能数**：11 个（10 个可选 + 1 个回血默认自带，不占技能栏）
- **对战模式**：
  - 本地对战（同设备双人）
  - 房间制 P2P 对战（一人创建房间，另一人输入房间码加入）

---

## 2. 游戏流程

```
主菜单
  ├── 本地对战 ───────────────┐
  ├── 创建房间（房主）────────┤
  ├── 加入房间（输入房间码）──┤
  └── 设置                    │
       ├── 音效开关/音量       │
       ├── 背景音乐开关/音量   │
       └── 胜负方式：一局定胜负 / 三局两胜
                              ↓
                        技能选择界面
                    （双方从 10 个技能中各选 3 个）
                              ↓
                        战斗场景
                    （回合制对战）
                              ↓
                        结算界面
                    （显示胜负结果）
                              ↓
                        返回主菜单
```

### 胜负方式规则
- **本地对战**：使用设置中的胜负方式
- **P2P 房间对战**：以房主的设置为准

### 三局两胜说明
- 每局结束后记录胜者
- 先赢 2 局者获胜
- 局与局之间回到技能选择界面（双方重新选技能）

## 3. 战斗场景概览

### 3.1 场景元素

- **石砖地面**：场景底部，承载地面站位点
- **三个黄色悬空平台**：从左到右排列，每个平台上有一个站位点（V8、V7、V6），正下方各有对应地面站位点
- **巨大悬空石柱**：场景右侧高处，顶端有一个站位点（V5），但并非画面最右
- **传送风阵**：画面最右侧地面，位于石柱右下方，不可站立，到达后自动传送到石柱顶端（V5）
- **场景背景**：浮空城堡风格

### 3.2 站位点总览（9 个位置）

| 编号 | 位置描述 | Y 层级 | 说明 |
|------|----------|--------|------|
| V1 | 画面最左侧，左下角地面 | 地面 | 同时也是 V8 的左下方 |
| V2 | V8 正下方地面，V1 右侧 | 地面 | |
| V3 | V7 正下方地面，V2 右侧 | 地面 | |
| V4 | V6 正下方地面，V3 右侧 | 地面 | |
| V5 | 石柱顶端，画面右侧高处 | 最高 | 非画面最右侧 |
| V6 | V5 左侧的平台 | 高台 | 黄色悬空平台（最右），V4 正上方 |
| V7 | V6 左下方的平台 | 中台 | 黄色悬空平台（中间），V3 正上方 |
| V8 | V7 左下方的平台 | 低台 | 黄色悬空平台（最左），V2 正上方 |
| V9 | 画面最右侧地面，V5 右下方 | 地面 | ⚠️ 传送风阵，到此自动传送到 V5 |

### 3.3 重要规则
- **V1 ~ V8**：可以站立、可以释放技能、可以攻击
- **V9**：**不可站立**，到达后立即传送到 V5。不能从 V5 跳到 V9
- 两名玩家**不能站在同一个站位点上**

## 4. 战斗规则

### 4.1 回合制
- 每个回合，双方各获得一次行动机会
- 行动包括：**移动** + **释放技能**（或选择不移动/不使用技能）
- 双方行动完毕后，进入下一回合
- 先手/后手：第一局随机，后续局由上一局败者先手

### 4.2 移动规则
- 每回合可移动 **1 个身位**（神速技能可持续 2 回合移动 2 个身位）
- 移动路线见 [站位地图设计文档](./站位地图设计.md)
- 到达 V9（传送风阵）→ 自动传送到 V5

### 4.3 跳跃掉血规则
从平台跳到地面会掉血（不占用移动，属于额外行动）：
| 跳跃路线 | 掉血量 |
|----------|--------|
| V6 → V4（地面） | 700 |
| V7 → V3（地面） | 200 |
| V8 → V2（地面） | 0（不掉血） |

### 4.4 攻击距离
以"身位"为单位计算，参照 [站位地图设计文档](./站位地图设计.md) 中的距离矩阵

### 4.5 胜负判定
- 一方血量降至 0 → 另一方获胜
- 三局两胜模式下，先赢 2 局者获胜

## 5. 网络架构（房间制 P2P）

### 5.1 创建房间
1. 房主点击"创建房间"
2. 服务器生成唯一房间码（如 6 位数字）
3. 房主将房间码分享给对手
4. 房主进入等待界面

### 5.2 加入房间
1. 对手点击"加入房间"
2. 输入房间码
3. 匹配成功后双方进入技能选择界面

### 5.3 通信方式
- 使用 **WebSocket** 进行实时通信
- 通信内容包括：选技能、移动、攻击、状态同步
- 房主端作为权威端处理游戏逻辑，客机端同步显示

### 5.4 设置规则
- P2P 对战时，房主的设置（胜负方式）覆盖客机的设置
- 音效/BGM 设置各自独立（本地设置，不影响对方）

---

## 6. 设置系统

### 6.1 设置项
| 设置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 音效开关 | 开关 | 开启 | 控制技能音效、攻击音效等 |
| 音效音量 | 滑块 | 80% | 0% ~ 100% |
| 背景音乐开关 | 开关 | 开启 | 控制战斗BGM、菜单BGM |
| 背景音乐音量 | 滑块 | 80% | 0% ~ 100% |
| 胜负方式 | 选择 | 一局定胜负 | 一局定胜负 / 三局两胜 |

### 6.2 存储方式
- 使用浏览器 `localStorage` 持久化保存设置
- 下次打开游戏自动加载上次设置

---

## 7. 排行榜系统

### 7.1 积分规则
- 每赢一场 P2P 对战 +10 分
- 本地对战不计分
- 积分累计，不扣分

### 7.2 排行榜
- 按积分降序排列
- 显示：排名、玩家名、积分、胜场数
- 数据存储在服务端数据库
