## 核心设计原则

### 1. 数据与逻辑分离
- `数据/` 目录下的文件**只存数据**，不写逻辑
- `系统/` 目录下的文件**只写逻辑**，从数据层读取配置
- 结果：新增技能/角色/地图 → 只需在 `数据/` 加条目，不动任何代码

### 2. 事件驱动解耦
- 各系统之间不直接互相调用
- 通过 `核心/事件总线.js` 通信
- 结果：修改技能系统不会影响移动系统

### 3. 场景独立
- 每个场景是独立模块，有自己的 init/destroy
- 场景切换时旧场景彻底销毁，不残留状态

## 如何新增内容

### 新增技能（最常用）
1. 打开 `数据/技能配置.js`
2. 在 `技能列表` 数组末尾添加一个技能对象
3. 完成。选择界面自动出现，战斗逻辑自动生效

```
{ 名称: '新技能', 类型: '伤害', 攻击距离: 1, 伤害: 500, 冷却: 3, ... }
```

### 新增角色
1. 打开 `数据/角色配置.js`
2. 添加角色对象
3. 在 `游戏资源/图像/人物/` 放入新角色素材
4. 在 `游戏资源/序列帧/人物动作/` 放入新角色动画
5. 在 `实体/` 新建角色类文件（可选，不建也能跑）

### 新增地图
1. 打开 `数据/地图配置.js`，注册新地图
2. 复制 `地图/站位数据.js` 为新文件，修改坐标和邻接关系
3. 在 `游戏资源/图像/场景/` 放入新背景图

## 注意
- 不要删除 `.workbuddy/` 文件夹，这是项目的 AI 记忆
- 所有资源路径使用相对于项目根目录的路径，方便部署
- Canvas 设计分辨率：1280 × 720，运行时等比缩放
